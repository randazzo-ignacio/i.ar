# WireGuard Setup Guide

## Topology: Hub and Spoke

```
                    randazzo-ar (Hub)
                    10.66.0.1
                    Public: 51820/udp
                         │
          ┌──────────────┼──────────────┐
          │              │              │
     ob-ar (spoke)  i-ar (spoke)   server-pc (spoke)
     10.66.0.2       10.66.0.3      10.66.0.5
          │              │              │
          └──────────────┴──────────────┘
                    laptop (spoke)
                    10.66.0.4
```

All spoke nodes connect to the hub. The hub routes traffic between peers. This means:
- ob-ar can reach i-ar (traffic goes: ob-ar → hub → i-ar)
- server-pc can reach laptop (traffic goes: server-pc → hub → laptop)
- The hub must have IP forwarding enabled

## Key Generation

```bash
# Generate keys for each host:
./scripts/generate-wg-keys.sh

# This creates wg-keys/ with:
#   randazzo-ar.private  randazzo-ar.public
#   ob-ar.private       ob-ar.public
#   i-ar.private        i-ar.public
#   laptop.private      laptop.public
#   server-pc.private   server-pc.public
#   vault-snippet.yml   (ready to paste into vault.yml)
```

## Configuration

All WireGuard config is generated by Ansible from templates. The template at `roles/wireguard/templates/wg0.conf.j2` produces:

### Hub (randazzo-ar):
```ini
[Interface]
PrivateKey = <hub-private-key>
Address = 10.66.0.1/16
ListenPort = 51820

# Peer: ob-ar
[Peer]
PublicKey = <ob-ar-public>
AllowedIPs = 10.66.0.2/32
Endpoint = <ob-ar-public-ip>:51820
PersistentKeepalive = 25

# Peer: i-ar
[Peer]
PublicKey = <i-ar-public>
AllowedIPs = 10.66.0.3/32
Endpoint = <i-ar-public-ip>:51820
PersistentKeepalive = 25

# ... (all peers listed)
```

### Spoke (e.g., ob-ar):
```ini
[Interface]
PrivateKey = <ob-ar-private>
Address = 10.66.0.2/16
ListenPort = 51820

# Peer: randazzo-ar (hub)
[Peer]
PublicKey = <hub-public>
Endpoint = <hub-public-ip>:51820
AllowedIPs = 10.66.0.0/16  # All WG traffic goes through hub
PersistentKeepalive = 25
```

## Management Commands

```bash
# Check status on any host:
wg show

# Restart WireGuard:
systemctl restart wg-quick@wg0

# Add a new peer (update vault.yml, re-run playbook):
ansible-playbook playbooks/wireguard.yml --vault-password-file .vault_pass

# Check handshake status:
wg show wg0 latest-handshakes

# Check transfer stats:
wg show wg0 transfer

# Troubleshoot:
journalctl -u wg-quick@wg0 -f
```

## Adding a New Peer

1. Generate keys: `wg genkey | tee /tmp/new.key | wg pubkey > /tmp/new.pub`
2. Add to `group_vars/vault.yml`:
   ```yaml
   wg_private_keys:
     new-host: "<private-key>"
   wg_public_keys:
     new-host: "<public-key>"
   ```
3. Add host to `inventory/hosts.yml` with `wg_ip: 10.66.0.6`
4. Create `host_vars/new-host.yml`
5. Re-run: `ansible-playbook playbooks/wireguard.yml --vault-password-file .vault_pass`

## Cloudflare Tunnel Fallback

If WireGuard is blocked (UDP filtering), use Cloudflare Tunnel:

1. Install `cloudflared` on your local machine
2. Connect: `cloudflared access ssh --hostname randazzo.net.ar`
3. This routes through Cloudflare's network to randazzo-ar

The tunnel is always active on randazzo-ar (runs as a systemd service). It provides:
- SSH access to the proxy server
- Can be extended to proxy other services (add public hostnames in Cloudflare dashboard)
