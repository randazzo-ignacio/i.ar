* Software RAID 1 + Borgbackup (USB External Drives)

Setup guide for creating a mirrored RAID array from two external USB drives and configuring automated encrypted backups with borgbackup.

** Prerequisites

- Two external USB drives of equal size
- Root access to the host machine
- =mdadm= and =borgbackup= installed

#+begin_src bash
  # Fedora
  dnf install mdadm borgbackup

  # Debian/Ubuntu
  apt install mdadm borgbackup
#+end_src

** Step 1: Identify the Drives

#+begin_src bash
  lsblk
#+end_src

Identify the two USB drives by their device names (e.g. =/dev/sdb=, =/dev/sdc=). Verify by checking disk size and manufacturer. **Double-check before proceeding -- the next step destroys all data on the target drives.**

** Step 2: Create the RAID 1 Array

#+begin_src bash
  # Replace sdX and sdY with your actual device names
  mdadm --create --verbose /dev/md0 --level=1 --raid-devices=2 /dev/sdX /dev/sdY
#+end_src

This creates a mirrored array. Both drives contain identical data. If one fails, the array continues operating in degraded mode.

** Step 3: Save the RAID Configuration

So the array is recognized on boot:

#+begin_src bash
  # Generate the config line
  mdadm --detail --scan >> /etc/mdadm.conf

  # Update initramfs so the array is available early in boot
  # Fedora
  dracut --force

  # Debian/Ubuntu
  update-initramfs -u
#+end_src

** Step 4: Format the Array

#+begin_src bash
  # ext4 (simple, reliable, widely supported)
  mkfs.ext4 /dev/md0

  # Alternatively, btrfs (built-in snapshots, compression, dedup)
  # mkfs.btrfs /dev/md0
#+end_src

** Step 5: Mount the Array

#+begin_src bash
  # Create mount point
  mkdir -p /mnt/nas

  # Mount immediately
  mount /dev/md0 /mnt/nas

  # Persist across reboots via fstab
  echo '/dev/md0 /mnt/nas ext4 defaults 0 2' >> /etc/fstab
#+end_src

For btrfs, replace =ext4= with =btrfs= and =defaults= with =defaults,compress=zstd=.

** Step 6: Initialize the Borg Backup Repository

#+begin_src bash
  # Create the encrypted backup repo on the NAS
  # You will be prompted to set a passphrase -- store it somewhere safe
  borg init --encryption=repokey /mnt/nas/borg-backup
#+end_src

** Step 7: Run the First Backup

#+begin_src bash
  borg create --stats /mnt/nas/borg-backup::'{hostname}-{now}' /root/.emacs.d
#+end_src

Verify the backup succeeded and check the archive list:

#+begin_src bash
  borg list /mnt/nas/borg-backup
#+end_src

** Step 8: Test a Restore

An untested backup is not a backup.

#+begin_src bash
  # Restore a single file to a temp location
  borg extract --dry-run /mnt/nas/borg-backup::LATEST path/to/file

  # Actually extract (into a temp directory)
  cd /tmp && borg extract /mnt/nas/borg-backup::LATEST path/to/file
#+end_src

** Step 9: Automate with a Systemd Timer

Create a backup service:

#+begin_src bash
  cat > /etc/systemd/system/borg-backup.service << 'EOF'
  [Unit]
  Description=Borg Backup
  Wants=borg-backup.timer

  [Service]
  Type=oneshot
  Environment=BORG_PASSPHRASE='your-passphrase-here'
  ExecStart=/usr/bin/borg create --stats /mnt/nas/borg-backup::'{hostname}-{now}' /root/.emacs.d
  ExecStartPost=/usr/bin/borg prune --keep-daily 7 --keep-weekly 4 --keep-monthly 6 /mnt/nas/borg-backup
  EOF
#+end_src

Create the timer:

#+begin_src bash
  cat > /etc/systemd/system/borg-backup.timer << 'EOF'
  [Unit]
  Description=Nightly Borg Backup

  [Timer]
  OnCalendar=*-*-* 03:00:00
  Persistent=true

  [Install]
  WantedBy=timers.target
  EOF
#+end_src

Enable and start:

#+begin_src bash
  systemctl daemon-reload
  systemctl enable --now borg-backup.timer
#+end_src

** Useful Borg Commands

#+begin_src bash
  # List all archives
  borg list /mnt/nas/borg-backup

  # Show contents of a specific archive
  borg list /mnt/nas/borg-backup::archive-name

  # Prune old archives (keep 7 daily, 4 weekly, 6 monthly)
  borg prune --keep-daily 7 --keep-weekly 4 --keep-monthly 6 /mnt/nas/borg-backup

  # Check repository integrity
  borg check /mnt/nas/borg-backup

  # Mount a repo as filesystem (browse old versions)
  borg mount /mnt/nas/borg-backup /mnt/borg-mount
  # Unmount when done
  borg umount /mnt/borg-mount
#+end_src

** RAID Maintenance

#+begin_src bash
  # Check array status
  cat /proc/mdstat
  mdadm --detail /dev/md0

  # Monitor failures (should be set up automatically on most distros)
  mdadm --monitor --scan --daemonise

  # Replace a failed drive
  mdadm /dev/md0 --remove /dev/sdX
  mdadm /dev/md0 --add /dev/sdY

  # Rebuild progress
  cat /proc/mdstat
#+end_src

** Notes

- RAID is not a backup. RAID protects against drive failure. Borg protects against accidental deletion, corruption, and user error. You need both.
- The =BORG_PASSPHRASE= environment variable in the systemd service is in plaintext. For better security, use a passphrase file with =BORG_PASSCOMMAND='cat /etc/borg-passphrase'= and restrict its permissions (=chmod 600=).
- USB external drives are slower than internal drives. Backup windows may be longer than expected. Run backups during low-activity hours.
- If a USB drive disconnects unexpectedly, the RAID array will degrade. Reconnect it and use =mdadm --add= to rebuild.
